# 产品评估指标框架 (Metrics Framework)

## 1. 指标框架概述
为了量化衡量“赛博朋克业务看板生成器”的产品价值和用户体验，我们采用 **HEART 模型**（Happiness, Engagement, Adoption, Retention, Task Success）结合 **AARRR 模型** 进行指标设计。

## 2. 北极星指标 (North Star Metric)
**指标名称**: **周活跃看板生成数 (Weekly Generated Dashboards)**
**定义**: 用户在过去7天内成功导出（下载 HTML/PNG/CSV）的看板总数。
**选择理由**: 该指标直接反映了产品为用户创造的核心价值——产出可用的分析结果。只有当用户觉得产品好用、生成结果满意时，才会执行“导出”这一最终动作。

## 3. 详细指标体系 (HEART 模型)

### 3.1 Happiness (愉悦度)
- **指标**: **NPS (净推荐值)**
  - *定义*: 用户问卷打分“你会向同事推荐这个工具吗？”(0-10分)。
  - *目标*: > 30 (SaaS工具基准)。
- **指标**: **导出后满意度评分**
  - *定义*: 导出成功后弹窗询问“对本次生成结果满意吗？”(1-5星)。
  - *目标*: > 4.5星。

### 3.2 Engagement (参与度)
- **指标**: **单次会话图表保存数**
  - *定义*: 平均每次会话中，用户点击“保存图表”的次数。
  - *目标*: > 3个 (说明用户不仅是试玩，而是生成了完整报告)。
- **指标**: **高级功能使用率**
  - *定义*: 使用“公式编辑器”或“多维筛选”的用户占比。
  - *目标*: > 20% (代表产品深度被挖掘)。

### 3.3 Adoption (采用度)
- **指标**: **新用户次日留存率**
  - *定义*: 新增用户在第二天再次访问的比例。
- **指标**: **上传转化率**
  - *定义*: 访问首页的用户中，完成文件上传动作的比例。
  - *目标*: > 60%。

### 3.4 Retention (留存度)
- **指标**: **周留存率**
  - *定义*: 上周使用过的用户，本周继续使用的比例。
  - *目标*: > 40% (工具类产品)。

### 3.5 Task Success (任务完成度) - **最关键**
- **指标**: **核心流程完成率**
  - *定义*: 上传文件 -> ... -> 导出文件 的全流程转化率。
  - *目标*: > 50%。
- **指标**: **报错率**
  - *定义*: 用户操作过程中触发系统 Error 的会话比例。
  - *目标*: < 1%。
- **指标**: **生成耗时**
  - *定义*: 从上传文件到第一个图表渲染出来的平均时长。
  - *目标*: < 3秒。

## 4. 功能级评估指标

| 功能模块 | 关键指标 | 成功阈值 | 监测方式 |
| :--- | :--- | :--- | :--- |
| **数据接入** | 文件解析成功率 | > 95% | 后台日志 |
| **指标公式** | 公式计算错误率 | < 5% | 错误捕获 |
| **图表交互** | 筛选响应延迟 | < 1s (P90) | 前端埋点 |
| **异常检测** | 异常点点击查看率 | > 10% | 点击热图 |
| **导出功能** | 导出文件下载完成率 | > 99% | 事件追踪 |

## 5. 指标监测计划
- **Phase 1 (MVP)**: 重点监测 **上传转化率** 和 **核心流程完成率**，确保流程跑通，无致命Bug。
- **Phase 2 (V1.0)**: 加入 **高级功能使用率** 和 **报错率** 监测，优化功能的稳定性和易用性。
- **Phase 3 (V2.0)**: 引入 **NPS** 和 **留存率**，关注长期价值和口碑。
